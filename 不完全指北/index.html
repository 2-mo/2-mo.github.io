            # 简介 脑启发 可信计算 场景分析

            提示

            视觉智能旨在利用计算机技术来模拟人类视觉系统的技术，当前已在特定场景中超越了人类视觉能力，但是依旧存在可解释性差、鲁棒性差和使用安全等问题，引发了信任危机。“可信”成为当下人工智能技术关注的焦点。借鉴大脑神经结构、认知功能与学习特性，研究脑启发的视觉感知与认知模型是解决可信问题的重要手段。本报告将具体介绍人类选择性注意力、上下文推理、联想记忆、反馈学习和多粒度感知等机制在可信视觉智能中的应用，结合脑启发和类人学习机制的视觉计算克服当前模型与方法的局限性，提升视觉计算的性能（准确性、鲁棒性、自适应性、可泛化性、可解释性等）。

            # 一、如何进入研究领域

            根据导师的研究方向或者项目的具体要求，找到一个感兴趣的研究方向（例如目标检测领域中的小目标检测、小样本检测、弱监督目标检测等）。看几篇研究方向相关综述（不一定必须是英文论文），一边看一边进行整理（可以借鉴一些中文的博客来进行总结），主要围绕以下几个问题进行整理：研究方向的概念、有什么应用、存在的挑战及问题、有哪些解决方案、针对该方向常用的数据集、现有方法的性能以及未来发展方向。这样整理就类似于在写一个小综述，后面可以作为毕业论文的第一二章。在整理过程中发现自己感兴趣的应用领域，然后找一下该领域上是否有可用的数据集，找数据集上的好论文，简单看他的思想，看有没有代码（paperswithcode上有根据数据集分类的论文，可以参考）。

            # 二、怎样找论文

            CV领域的几个顶会：CVPR/ICCV/ECCV/AAAI/TIP/TPAMI。从这些顶会里找几篇高质量论文，对每篇论文进行自己的总结，总结格式：本文针对xxx问题，xxx通过xxx，并xxx，从而xxx（达到了什么样的效果）。最好按照时间顺序看。不用精读全文，主要是要找出一篇文章具体解决了什么问题，然后能用自己的话总结出来。

            # 三、怎样看论文

            重点是找对应关系，找问题。一篇好的文章是有很明确的对应关系的，abstrat和introduction基本上是一一对应的，通常会先介绍领域挑战，然后讲本文所针对的问题，针对该问题提出了什么方法，最后达到了什么样的效果。（通常好文章的图有很丰富的信息，可以通过看图就明确文章重点解决的问题）。带着以下几个问题读论文：

            1. 这篇论文主要是为了解决什么问题？
            2. 论文的主要创新点：这个创新点的依据是什么？基于什么考虑（即动机）？
            3. 整体的网络结构是什么样的？
            4. 自己的思考：看完一篇文章后思考自己的收获，不一定是要从中看出idea，更多的是去学习别人的写作方式、绘图、配色等，其实就是去发现值得自己学习的地方。

            # 四、感受论文

            通过看的论文总结出好论文的共性，然后在New Tab（arxiv-sanity-lite.comopen in new window）上尝试根据自己的感觉找出可能是好论文的文章。

            # 五、怎么讲论文

            看懂论文后要能将它讲出来，人的思维是网状的，但是论文限于形式是线性的，有个转换过程，所以需要能讲清楚。具体汇报流程如下：

            1. 发表在哪，作者是谁，是哪个机构/团队做的（告诉大家这个质量很高，认真听）

            2. 论文是什么领域，在哪个数据库，有什么应用（与大家有什么联系，阐述研究意义）

            3. 当前领域存在的挑战是什么，论文解决的是什么问题（核心！问题讲清楚大家才能听懂）

            4. 简要介绍基本概念及相关工作（汇报需要所有人可以听懂）

            5. 论文的动机是什么，主要贡献是什么，基于什么做出了改进（优化思路）

            6. 评价标准是什么，取得了什么效果，和当前SOTA方法对比怎么样

            7. 实验结果如何验证其改进效果，是否解决其针对的问题

            8. 自己的收获，论文值得学习的地方，其解决的问题在自己的领域是否存在，提出方法的思路是否可以模仿迁移，实现方式是否可以参考套用，论文的表现形式是否可以学习借鉴
